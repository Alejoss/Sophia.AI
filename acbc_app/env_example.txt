================================================================================
IMPORTANT NOTES
================================================================================

1. SECURITY:
   - NEVER commit the .env file to version control
   - Always use strong, unique passwords in production
   - Generate a secure SECRET_KEY for production
   - Keep your .env file permissions restricted (chmod 600)

2. PRODUCTION REQUIREMENTS:
   - ENVIRONMENT must be set to PRODUCTION
   - DEBUG must be False
   - ALLOWED_HOSTS must include your actual domain(s)
   - ACADEMIA_BLOCKCHAIN_SKEY must NOT be the default development key
   - Database credentials must be secure

3. VALIDATION:
   - The application will raise errors if required production variables are missing
   - DEBUG=True will cause an error in PRODUCTION environment
   - ALLOWED_HOSTS is required in PRODUCTION environment
   - If SEND_EMAILS=True in PRODUCTION, POSTMARK_SERVER_TOKEN is required

4. GENERATING SECRET KEY:
   Run this command to generate a secure secret key:
   python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

5. DOCKER COMPOSE:
   - When using docker-compose, DB_HOST should be "postgres" (the service name)
   - POSTGRES_* variables are used by the postgres container
   - DB_* variables are used by the Django application

6. VARIABLE NAMING:
   - All variable names are case-sensitive
   - No spaces around the = sign
   - No quotes needed for values (unless the value itself contains special characters)
   - Comments start with # and must be on their own line
